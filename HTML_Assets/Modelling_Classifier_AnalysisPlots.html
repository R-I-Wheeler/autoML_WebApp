Classification - Model Analysis Visualisations

The following plots may be used to analyse the performance of the model.

Receiving Operating Charachteristic/Area Under the Curve

The Receiver Operating Characteristic (ROC) is a measure of a classifier’s predictive quality that compares and visualizes the tradeoff between the model’s sensitivity and specificity. The ideal point is therefore the top-left corner of the plot: false positives are zero and true positives are one.

Classification Report

Precision - for all instances classified as positive, what percentage were correct.

Recall - for all instances that are actually positive, what percentage were classified correctly.

F1 Score - a weighted harmonic mean of Precision and Recall, the best score is 1.0, the worst is 0.0

Support - the number of actual occurences of the class within the dataset

Confusion Matrix

Shows a comparison between the test values predicted classes and their actual classes.

Feature Importance

The features are ranked according to the explained variance each feature contributes to the model.

Learning Curve

If the training and cross-validation scores converge together as more data is added, then the model will probably not benefit from more data. If the training score is much greater than the validation score then the model probably requires more training examples in order to generalize more effectively. Variability during cross-validation is shown with the shaded areas that represent a standard deviation above and below the mean for all cross-validations. If the model suffers from error due to bias, then there will likely be more variability around the training score curve. If the model suffers from error due to variance, then there will be more variability around the cross validated score.

Precision Recall Curve

Shows the tradeoff between a classifier’s precision, a measure of result relevancy, and recall, a measure of completeness..
